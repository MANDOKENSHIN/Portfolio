# 用語集

**要件ID**: GLOSSARY-001

| 用語 | 定義 |
|-----|------|
| タイマー | 設定した時間からカウントダウンし、0になったら通知を発行する機能 |
| アラーム | 指定した時刻になったら通知を発行する機能 |
| ストップウォッチ | 開始からの経過時間を計測する機能 |
| カウントダウン | 設定した時間から0に向かって時間を減らしていくこと |
| ラップタイム | ストップウォッチで記録した、ある時点での経過時間 |
| スプリットタイム | 前回のラップタイムから次のラップタイムまでの経過時間 |
| モード | タイマー、アラーム、ストップウォッチのいずれかの機能状態 |
| バックグラウンド動作 | 他のモードに切り替えても、機能が継続して動作すること |
| ローカルストレージ | ブラウザにデータを保存する仕組み（localStorage API） |
| レスポンシブデザイン | 画面サイズに応じてレイアウトを最適化するデザイン手法 |
| ダークモード | 背景を黒系、テキストを白系にした配色モード |
| ライトモード | 背景を白系、テキストを黒系にした配色モード |
| ブレークポイント | レスポンシブデザインにおいて、レイアウトが切り替わる画面幅の境界値 |
| プリセット | あらかじめ設定された値。タイマーでよく使う時間（1分、5分等）を素早く設定する機能 |
| スヌーズ機能 | アラーム通知を一時的に停止し、指定時間後（5分後等）に再度通知する機能 |
| バリデーション | 入力データが正しい形式・範囲であるかを検証すること |
| エラーハンドリング | エラーが発生した際に、適切に処理して異常終了を防ぐこと |
| モーダル表示 | 画面上に重ねて表示されるダイアログやウィンドウ。背景操作をブロックする |
| オーバーレイ | モーダル表示の背景に表示される半透明の層 |
| トグルスイッチ | オン/オフを切り替えるスイッチ型のUI部品 |
| WCAG | Web Content Accessibility Guidelines（Webアクセシビリティのガイドライン） |
| アクセシビリティ | 高齢者や障害者を含む、すべての人が利用できるようにする配慮 |
| コントラスト比 | テキストと背景の明度差の比率。視認性の指標 |
| Notification API | ブラウザのネイティブ通知を表示するWeb API |
| JSON | JavaScript Object Notation。データ交換フォーマットの一種 |
| TypeScript | JavaScriptに型システムを追加したプログラミング言語 |
| ESLint | JavaScriptのコード品質をチェックするツール |
| Prettier | コードフォーマッター。コードスタイルを自動整形するツール |
| コードレビュー | 他の開発者がコードを確認し、品質を向上させるプロセス |
| 単体テスト | 個々の関数やモジュール単位で行うテスト |
| 統合テスト | 複数のモジュールを組み合わせて行うテスト |
| システムテスト | システム全体の動作を確認するテスト |
| 受け入れテスト | ユーザー視点で要件を満たしているかを確認するテスト |
| テストカバレッジ | テストによって実行されたコードの割合 |
| ブラウザ互換性 | 異なるブラウザ（Chrome、Firefox等）で正常に動作すること |
| Polyfill | 古いブラウザで新しいAPI機能を使えるようにするコード |
| フリー素材 | 著作権フリーまたは利用許諾された音声・画像ファイル |
| マイルストーン | プロジェクトの重要な節目や達成目標 |
| ステークホルダー | プロジェクトに関わる利害関係者 |
| ペルソナ | 想定ユーザーの具体的な人物像 |
| ユースケース | システムの具体的な使用シナリオ |
| 非機能要件 | 性能、セキュリティ等、機能以外のシステム要件 |
| 制約条件 | プロジェクトにおける技術的・予算的・時間的な制限 |

- 一般的なIT用語で自明なもの（ブラウザ、HTML、CSS等）は省略しています
